# Проект: "Alena" — Менеджер задач
## Обзор
"Alena" — это личный менеджер задач, ориентированный на интуитивное управление приоритетами через Drag & Drop.

## Технологический стек
- **Frontend:** Next.js 14+ (App Router), Tailwind CSS, TypeScript.
- **Backend/DB:** Supabase (PostgreSQL).
- **Drag & Drop:** `@dnd-kit/core`, `@dnd-kit/sortable`, `@dnd-kit/utilities`.
- **Icons:** `lucide-react`.

## Структура базы данных (Таблица `tasks`)
- `id` (bigint, PK) — ID задачи.
- `title` (text) — Заголовок.
- `is_completed` (boolean) — Статус выполнения.
- `is_today` (boolean) — Принадлежность к колонке "Задачи на сегодня".
- `display_order` (integer) — Порядок отображения в колонке.
- `created_at`, `deadline` — Временные метки.

## Правила интерфейса (UI/UX)
1. **Брендинг:** Заголовок приложения — "Alena".
2. **Колонки:** - "Все задачи" (бэклог, `is_today: false`).
   - "Задачи на сегодня" (`is_today: true`).
3. **Управление:** - Перетаскивание — основной метод управления.
   - Звездочки упразднены.
   - Редактирование осуществляется через иконку карандаша (Pencil) в каждой строке.
4. **Стиль:** Минимализм в духе Notion/Asana. Использование Tailwind для отступов и типографики.

## Архитектура Drag & Drop (Критически важно)
- **Контейнеры:** Используется двухуровневая система `SortableContext` с общим `DndContext`.
- **Синхронизация:** - Любое перемещение должно вызывать `moveTask` (Server Action).
   - **Оптимистичный UI:** Интерфейс обновляется мгновенно, не дожидаясь ответа сервера.
   - **Rollback:** Если запрос к Supabase вернул ошибку, состояние должно быть принудительно возвращено к предыдущему (snapshot).
- **Функция moveTask(id, is_today, new_order):** Должна атомарно обновлять статус и позицию в базе данных.

## Инструкция для ИИ (Cursor/Gemini)
- При внесении любых правок в `page.tsx` или `actions/tasks.ts` соблюдай принцип атомарности.
- Никогда не ломай существующий Drag & Drop.
- При добавлении новых функций (например, дедлайнов) сохраняй текущую семантику и стилистику.
